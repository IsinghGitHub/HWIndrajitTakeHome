  ## What exists today
  Two independent scripts that:
  1. Each read the same IND parquet files + ORG CSV files from disk (serial, per-file)
  2. Each join the same two score CSVs
  3. Both compute common/unique NPI flags via `groupby.transform('nunique')`
  4. Count script: aggregates into 3 CSV tables; Download script: writes per-provider CSV+parquet
  5. Both silently swallow all exceptions

  ## Architecture to implement (from solution_approach_map.md)
  - **DuckDB core + Polars I/O**: DuckDB does all analytical work (pushdown filter on parquet, SQL joins, window functions for flags, GROUP BY for aggregation). Polars writes output files.
  - **One job, two output builders**: Single read+clean+score-join pass, branching at the end into count builder (→ 3 CSV tables) and download builder (→ CSV + parquet).
  - **Single data load**: IND and ORG files read ONCE, not twice.

  ## Bugs to fix (from solution_approach_map.md)
  1. **Dead empty-guard**: In `load_compare_files`, `if compare_ind_df.empty` assigns to `base_ind_df` not `compare_ind_df`. Fix the variable name.
  2. **NameError on empty compare list**: `final_data_ind`/`final_data_org` only bound inside the for-loop, used after. Guard with early return if list is empty.

  ## DuckDB implementation details

  **Loading IND parquet for a network:**
  ```python
  def _ind_paths(file_path, filespath, net_ids):
      paths = []
      for net in net_ids:
          d = f"{file_path}{filespath}/network_files/Individual/{net}"
          paths += [f"{d}/{f}" for f in os.listdir(d) if f not in SPEC_CODES]
      return paths

  def load_ind(con, file_path, filespath, net_ids, label, network_id_str, network_name, county_sql):
      paths = _ind_paths(file_path, filespath, net_ids)
      if not paths:
          con.execute(f"CREATE OR REPLACE TABLE ind_{label}(npi BIGINT) AS SELECT * FROM (VALUES (NULL::BIGINT)) WHERE false")
          return
      paths_sql = "[" + ",".join(f"'{p}'" for p in paths) + "]"
      con.execute(f"""
          CREATE OR REPLACE TABLE ind_{label} AS
          SELECT npi::BIGINT, type, presentation_name, first_name, last_name,
                 '{network_id_str}' AS network_id, '{network_name}' AS network_name,
                 "Address First Line", "Parent_Organization" AS parent_org,
                 location_confidence, location_confidence_address_level,
                 specialty_category, specialty_name, specialty_subspecialty,
                 specialty_code::VARCHAR AS specialty_code, "Specialty Flag",
                 "New Flag", city, state, zip_code::VARCHAR AS zip_code, county_name,
                 "FIPS State County Code"::BIGINT AS fips,
                 "Specialty Description", "Is_Dual_Provider",
                 "Affiliation ID"::VARCHAR AS group_tin,
                 "Affiliation Presentation Name" AS group_name
          FROM read_parquet({paths_sql}, union_by_name=true)
          WHERE "FIPS State County Code"::BIGINT IN ({county_sql})
            AND location_confidence IN ('Low','Medium','High','LOCATION CONFIDENCE NOT AVAILABLE')
            AND "New Flag" IN ('PCP','Physician Specialists','Other Providers')
      """)
  ```

  **Loading ORG CSV for a network:**
  ```python
  def load_org(con, file_path, filespath, net_ids, label, network_id_str, network_name, county_sql):
      parts = []
      for net in net_ids:
          org_file = f"{file_path}{filespath}/network_files/Organization/{net}.csv"
          try:
              parts.append(f"""
                  SELECT npi::BIGINT, type, presentation_name,
                         first_name::VARCHAR AS first_name, last_name::VARCHAR AS last_name,
                         '{network_id_str}' AS network_id, '{network_name}' AS network_name,
                         carrier_name, specialty_category, specialty_name, specialty_subspecialty,
                         specialty_code::VARCHAR AS specialty_code,
                         "New Flag", city, state, zip_code::VARCHAR AS zip_code, county_name,
                         "FIPS State County Code"::BIGINT AS fips,
                         "Address First Line", Status,
                         specialty_name AS "Specialty Description",
                         carrier_name AS parent_org,
                         NULL::VARCHAR AS group_tin, NULL::VARCHAR AS group_name
                  FROM read_csv('{org_file}', auto_detect=true)
                  WHERE UPPER(Status) = 'ACTIVE'
                    AND "FIPS State County Code"::BIGINT IN ({county_sql})
                    AND "New Flag" != 'None'
              """)
          except Exception:
              pass
      if parts:
          union = " UNION ALL ".join(f"({p})" for p in parts)
          con.execute(f"CREATE OR REPLACE TABLE org_{label} AS {union}")
      else:
          con.execute(f"CREATE OR REPLACE TABLE org_{label}(npi BIGINT) AS SELECT * FROM (VALUES (NULL::BIGINT)) WHERE false")
  ```

  **Score join for IND (DuckDB SQL):**
  The merge key is: `CASE WHEN "New Flag" = 'PCP' THEN 'PCP' ELSE "Specialty Description" END`
  ```sql
  CREATE OR REPLACE TABLE scored_ind AS
  SELECT i.*,
         s.cost_score, s.quality_score, s.npi_score,
         s.ma_util_score, s.ffs_util_score, s.quality_color
  FROM combined_ind i
  LEFT JOIN ind_scores s
    ON i.npi = s.npi
   AND i."New Flag" = s."New Flag"
   AND i.fips = s.fips
   AND CASE WHEN i."New Flag" = 'PCP' THEN 'PCP' ELSE i."Specialty Description" END = s.specialty_desc
  ```

  **Score join for ORG (DuckDB SQL):**
  ```sql
  CREATE OR REPLACE TABLE scored_org AS
  SELECT o.*,
         s.quality_score, s.cost_score, s.ffs_util_score, s.ma_util_score,
         NULL::FLOAT AS npi_score, NULL::VARCHAR AS quality_color
  FROM combined_org o
  LEFT JOIN org_scores s
    ON o.npi = s.npi
   AND UPPER(TRIM(o."Address First Line")) = s.address
   AND o.state = s.state
   AND o.fips = s.fips
  ```

  **Common/unique flags for COUNT output (window function):**
  ```sql
  SELECT *,
    CASE WHEN COUNT(DISTINCT network_id) OVER (
      PARTITION BY npi, state, fips, "New Flag"
    ) > 1 THEN 'common' ELSE 'unique' END AS common_unique
  FROM combined_ind
  ```

  **Y/N membership flags for DOWNLOAD output (window function):**
  ```sql
  SELECT *,
    CASE WHEN COUNT(DISTINCT network_id) OVER (
      PARTITION BY npi, state, county_name, "New Flag"
    ) > 1 OR network_id = '{base_id}' THEN 'Y' ELSE 'N' END AS base_flag,
    CASE WHEN COUNT(DISTINCT network_id) OVER (
      PARTITION BY npi, state, county_name, "New Flag"
    ) > 1 OR network_id = '{compare_id}' THEN 'Y' ELSE 'N' END AS compare_flag
  FROM combined_ind
  ```

  **Score categorization for COUNT (replacing groupby.apply with FILTER aggregation):**
  ```sql
  -- 3 rows per group (High, Medium, Low) via UNION ALL
  SELECT "New Flag", network_name, network_id,
    COUNT(*) FILTER (WHERE quality_cat='High') AS "Quality Score",
    COUNT(*) FILTER (WHERE cost_cat='High') AS "Cost Score",
    COUNT(*) FILTER (WHERE util_cat='High') AS "Utilization Score",
    'High' AS "Scoring Category"
  FROM scored_with_cats GROUP BY "New Flag", network_name, network_id
  UNION ALL
  SELECT "New Flag", network_name, network_id,
    COUNT(*) FILTER (WHERE quality_cat='Medium') AS "Quality Score",
    COUNT(*) FILTER (WHERE cost_cat='Medium') AS "Cost Score",
    COUNT(*) FILTER (WHERE util_cat='Medium') AS "Utilization Score",
    'Medium' AS "Scoring Category"
  FROM scored_with_cats GROUP BY "New Flag", network_name, network_id
  UNION ALL
  SELECT "New Flag", network_name, network_id,
    COUNT(*) FILTER (WHERE cost_cat='Low') AS "Quality Score",
    COUNT(*) FILTER (WHERE cost_cat='Low') AS "Cost Score",
    COUNT(*) FILTER (WHERE util_cat='Low') AS "Utilization Score",
    'Low' AS "Scoring Category"
  FROM scored_with_cats GROUP BY "New Flag", network_name, network_id
  ```

  Score categorization helper view:
  ```sql
  CREATE OR REPLACE VIEW scored_with_cats AS
  SELECT *,
    CASE WHEN quality_score >= 4 THEN 'High'
         WHEN quality_score > 2 THEN 'Medium'
         WHEN quality_score <= 2 THEN 'Low'
         ELSE 'Not Available' END AS quality_cat,
    CASE WHEN cost_score >= 4 THEN 'High'
         WHEN cost_score > 2 THEN 'Medium'
         WHEN cost_score <= 2 THEN 'Low'
         ELSE 'Not Available' END AS cost_cat,
    CASE WHEN ma_util_score >= 80 THEN 'High'
         WHEN ma_util_score <= 40 THEN 'Low'
         WHEN ma_util_score > 40 THEN 'Medium'
         ELSE 'Not Available' END AS util_cat
  FROM scored_ind_all -- or scored_org_all
  ```
